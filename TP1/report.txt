1-1 It is better to run the container with a flag -e because, with this, all the passwords and others sensitive data are only available to the running container. So there is no risk of leaks.


docker run --network app-network --link runtp1:db -p 8080:8080 adminer
docker run -p 8888:5000 -e POSTGRES_DB=db -e POSTGRES_USER=usr -e POSTGRES_PASSWORD=pwd -v volume-name:/var/lib/postgresql/data --network app-network --name runtp1 leonletouriste/tp1
